Scope of var, let, and const

- `var` has function scope or global scope if declared outside functions. It is hoisted to the top of its containing function or script.

- `let` and `const` have block scope, meaning they are limited to the block of code (inside `{}`) where they are declared. They are not hoisted and do not have global scope by default.

The use cases of null and undefined

- `null` is used to explicitly indicate the absence of a value or to clear/reset a variable or object property.

- `undefined` is the default value for variables that have been declared but not assigned a value. It's also returned when accessing non-existent object properties or when fewer arguments are provided to a function than declared parameters.

What does mean by REST API?

A REST API is like a menu in a restaurant. It defines a set of rules for how you can order and interact with items (resources) on the menu (server). You use standard actions like "GET" (read), "POST" (add), "PUT" (update), and "DELETE" (remove) to interact with these items, which are identified by unique names (URLs). It's a way for software applications to talk to each other over the internet, similar to how you order food from a menu.
